# GitHub Archive Dashboard

A self-contained Rust application that imports your Google Sheets `.sql` export
files into a local SQLite database and serves a full GitHub-themed dashboard UI.

## Quick Start

```bash
# Build (requires Rust 1.75+)
cargo build --release

# Create the exports directory and drop your .sql files in
mkdir exports
cp /path/to/github_archive_*.sql exports/

# Run: imports any new .sql files, then starts the web server
./target/release/github-archive

# Open the dashboard
open http://localhost:3000
```

## Options

```
  -i, --import-dir <DIR>    Directory with .sql export files  [default: ./exports]
  -d, --database <FILE>     SQLite database path              [default: ./github_archive.db]
  -p, --port <PORT>         Web UI port                       [default: 3000]
      --import-only         Import files only, skip web server
```

## How It Works

1. **Import** — On startup, the app walks `--import-dir` for `*.sql` files.
   Each file's SHA-256 is checked against the `_imported_files` table.
   Files not yet imported are executed against the database, then recorded.
   Re-running is always safe — already-imported files are silently skipped.

2. **Serve** — An Axum web server starts on `--port`, serving the embedded
   single-page dashboard and a set of JSON API endpoints.

## API Endpoints

| Endpoint                    | Description                           |
|-----------------------------|---------------------------------------|
| `GET /api/overview`         | Global KPIs and group counts          |
| `GET /api/snapshots`        | List of all snapshot timestamps       |
| `GET /api/repos`            | All repos for a snapshot              |
| `GET /api/repo/:owner/:name`| Full detail for a single repo         |
| `GET /api/activity`         | 28-day activity window per repo       |
| `GET /api/backlog`          | Backlog health (age buckets) per repo |
| `GET /api/commits/weekly`   | 12-week commit series                 |
| `GET /api/contributors`     | Top contributors per repo             |
| `GET /api/pulls`            | Pull requests                         |
| `GET /api/groups`           | Per-group aggregated stats            |
| `GET /api/momentum`         | Net issue delta momentum table        |
| `GET /api/imported-files`   | Import log                            |

All data endpoints accept `?snapshot_ts=<ISO>` to query a historical snapshot.

## Dashboard Views

- **Overview** — KPI tiles, group cards, commit sparkline, snapshot history chart
- **Repositories** — Sortable full repo table with stars/forks/issues
- **Activity** — 28-day opened/closed breakdown with bar chart
- **Backlog Health** — Age-bucket donut chart + per-repo age distribution bars
- **Commits** — 12-week aggregate line chart + per-repo bar
- **Contributors** — Top contributors bar chart + detail table
- **Pull Requests** — Filterable PR list with state badges
- **Momentum** — Net issue delta bar chart (red = growing backlog, green = shrinking)
- **Groups** — CF / BOSH / Services side-by-side cards + stars comparison chart
- **Imported Files** — Full import log with SHA-256 hashes

## Database Schema

See the `.sql` files generated by the Google Sheets script for full schema.
The app adds one extra table: `_imported_files (filename, sha256, imported_at)`.
